# 贡献指南

感谢您对 Ac-Wiki 的关注！我们欢迎所有形式的贡献。

## 如何贡献？

### 1. 报告问题

如果您发现了内容错误、链接失效或其他问题，请通过以下方式报告：

- 在 GitHub 上提交 Issue
- 发送邮件到我们的邮箱
- 在 QQ 群中反馈

### 2. 内容贡献

我们欢迎您贡献以下类型的内容：

#### 校园生活经验
- 新生入学指南
- 选课经验分享
- 校园生活小贴士
- 学生服务使用指南

#### 学术资源
- 学习方法总结
- 考试备考攻略
- 学术工具推荐
- 学习资源整理

#### 技能教程
- 计算机基础教程
- 网络安全知识
- 生活技能分享
- 实用工具介绍

#### 成长规划
- 职业规划建议
- 实习就业经验
- 考研留学攻略
- 个人发展建议

### 3. 贡献流程

1. **Fork 项目**：在 GitHub 上 Fork 本项目
2. **创建分支**：为你的贡献创建一个新分支
3. **编写内容**：按照我们的格式规范编写内容
4. **提交 PR**：提交 Pull Request 并描述你的贡献
5. **等待审核**：我们会尽快审核你的贡献

### 4. 内容规范

#### Markdown 格式
- 使用标准的 Markdown 语法
- 标题使用 `#` 到 `####`
- 代码块使用 ``` 包围
- 图片使用相对路径

#### 内容要求
- 内容准确、实用
- 语言简洁明了
- 适当使用表情符号增加可读性
- 提供具体的操作步骤或建议

#### 文件命名
- 使用小写字母和连字符
- 文件名要有描述性
- 避免使用中文文件名

### 5. 示例

```markdown
# 标题

## 简介

这里是内容的简介...

## 详细内容

### 步骤一

1. 第一步
2. 第二步
3. 第三步

### 注意事项

- 注意点 1
- 注意点 2

## 相关链接

- [相关资源 1](链接)
- [相关资源 2](链接)
```
